# Simplified Agent Connection Guide for StudyPodLM

## For Non-Developers: How to Enable Agent Communication

If you're a StudyPodLM user (not a developer) and want to connect an AI agent (like myself) to your notebooks for collaborative learning, here's the simplest possible flow:

### 🚀 Ideal One-Click Connection Flow (What Should Exist)

**Step 1: In Your StudyPodLM Dashboard**
- Navigate to any notebook you want to share with an agent
- Click the "🤖 Agent Access" button in the notebook header (or Profile Menu)
- Click "Generate Agent Connection Package"

**Step 2: Download & Configure**
- Download a pre-configured `.zip` file containing:
  - `agent_skill/` folder with all necessary scripts
  - `CONFIG.json` with your notebook ID and permissions pre-filled
  - `README.md` with simple instructions
- Extract the zip to a folder on your computer

**Step 3: Launch the Agent**
- Double-click `start_agent.bat` (Windows) or run `./start_agent.sh` (Mac/Linux)
- Follow the simple prompts (if any)
- The agent will automatically:
  1. Authenticate using your credentials (securely)
  2. Connect to your specified notebook
  3. Begin monitoring for collaboration opportunities

### 🔧 Current Manual Flow (What Exists Now)

If the ideal one-click flow isn't available yet, here's how to do it with minimal technical effort:

**Prerequisites:**
- You must be logged into StudyPodLM via the web UI first
- Have Node.js installed (download from nodejs.org if needed)

**Steps:**
1. **Get Pairing Code**
   - In StudyPodLM UI: Profile Menu → Agent Pairing → "Generate Pairing Code"
   - Copy the 6-digit code shown

2. **Set Up Agent Environment**
   - Create a folder for your agent (e.g., `studypod-agent`)
   - Open terminal/command prompt in that folder
   - Run: `npm init -y` (creates basic package.json)
   - Run: `npm install axios` (installs required dependency)

3. **Create Agent Script**
   - Create a file named `agent.js` with this content:
   ```javascript
   const axios = require('axios');
   const fs = require('fs');
   
   // REPLACE WITH YOUR 6-DIGIT CODE FROM STEP 1
   const PAIRING_CODE = 'YOUR_CODE_HERE'; 
   
   // Optional: Customize these
   const AGENT_LABEL = 'My StudyPod Agent';
   const API_BASE = 'http://localhost:3001/api'; // Adjust if using different URL
   
   async function connectAgent() {
     try {
       console.log(`🔗 Pairing with code [${PAIRING_CODE}]...`);
       
       const response = await axios.post(`${API_BASE}/auth/pair/complete`, {
         code: PAIRING_CODE,
         label: AGENT_LABEL
       });
       
       const { key } = response.data;
       console.log('✅ Pairing successful!');
       console.log(`🔑 Your API Key: ${key}`);
       
       // Save for future use
       fs.writeFileSync('.env.agent', `STUDYPOD_API_KEY=${key}\nSTUDYPOD_API_URL=${API_BASE}\n`);
       console.log('💾 Configuration saved to .env.agent');
       
       // Test connection
       const notebooks = await axios.get(`${API_BASE}/notebooks`, {
         headers: { 'Authorization': `Bearer ${key}` }
       });
       
       console.log(`📚 Found ${notebooks.data.length} notebooks`);
       console.log('🎉 Agent is ready to collaborate!');
       
     } catch (error) {
       console.error('❌ Connection failed:', error.response?.data?.error || error.message);
     }
   }
   
   connectAgent();
   ```

4. **Run Your Agent**
   - In terminal: `node agent.js`
   - Replace `YOUR_CODE_HERE` with the 6-digit code from Step 1
   - Press Enter and follow any prompts

**💬 Interactive Learning**

**📊 Study Optimization**

### 🛠️ For Developers: Creating Reusable Agent Skills

If you want to create a shareable agent skill (like the GitHub repo you mentioned), here's what to include:

**Repository Structure:**
```
studypod-agent-skill/
├── src/
│   ├── index.js          # Main agent logic
│   ├── auth.js           # Handles authentication
│   └── notebook.js       # Notebook interaction methods
├── scripts/
│   ├── setup.sh          # Installation script
│   └── update-notebook.js # Example utility
├── .env.example          # Template for configuration
├── README.md             # Clear usage instructions
└── package.json          # Dependencies and scripts
```

**Key Features to Include:**
1. **Zero-Config Setup**: `npm install` + `npm run setup` should work
2. **Secure Credential Handling**: Never store raw credentials in repo
3. **Notebook Discovery**: Auto-find or let user specify target notebook
4. **Permission Scoping**: Limit agent to specific notebooks/actions
5. **Error Recovery**: Handle network issues, expired tokens gracefully
6. **Logging**: Clear output showing what the agent is doing

**Example Usage in README:**
```bash
# 1. Install the skill
npm install @yourusername/studypod-agent-skill

# 2. Run setup (guides you through pairing)
npx studypod-agent-setup

# 3. Start collaborating
npx studypod-agent --notebook "My Biology Notes" --action summarize
```

### 🔒 Security Considerations

For your protection as a user:
- Agents only get access to notebooks you explicitly authorize
- All agent actions are logged and attributable ("AGENT" badge)
- You can revoke agent access anytime from Profile Menu → API Keys
- Agent notes sync to EverMemOS for long-term persistence but respect your privacy settings
- No agent can access your account credentials or other notebooks without explicit permission

### 💡 Pro Tips for Best Collaboration

1. **Start Small**: Begin with one notebook to test the agent's capabilities
2. **Be Specific**: Give agents clear instructions like "Summarize this PDF in 3 bullet points"
3. **Review Contributions**: Periodically check agent-added notes for accuracy
4. **Combine Strengths**: Use agents for research/summarization, humans for critical thinking and creativity
5. **Iterate**: Adjust your requests based on what the agent does well vs. poorly

## Summary: The Simplest Possible Flow

For a non-developer user wanting to connect an agent like me to StudyPodLM:

1. **You click one button in the UI** → "Connect Agent to This Notebook"
2. **System generates a secure connection package** → Download it
3. **You run one command** → `./start-agent` (or double-click)
4. **Agent appears in your notebook** → Ready to collaborate

This eliminates all technical barriers while maintaining security and giving you full control over what the agent can access and do.

Until such a one-click flow exists, the manual process above (getting pairing code → running simple node script) is the most straightforward method available today.
